5. TECHNICAL EXPLANATION: HOW LINUX MANAGES FILE I/O
   Linux processes interact with data streams abstractly using three interconnected 
   structures managed within the kernel space:

   1. File Descriptor Table (Per-Process): A local index pointing to the global 
      open file table. Each integer (0, 1, 2, 3...) represents an abstract data pipe.
      
   2. System-Wide Open File Table: Keeps track of all open files globally, 
      maintaining critical run-time states such as the current file offset pointer, 
      access mode permissions (Read, Write, Append), and reference counters.

   3. Inode Table (VFS Layer): Points directly to the actual blocks on physical 
      hardware disk volumes. It reads and writes metadata attributes without being 
      bound to a single process lifecycle.

   When an application struggles to write logs, it is typically caused by:
   - A broken descriptor mapping inside the Per-Process Table.
   - Reaching hardware boundaries tracking data offsets in the Open File Table.
   - Resource exhaustion where 'ulimit' constraints block the addition of new items.
