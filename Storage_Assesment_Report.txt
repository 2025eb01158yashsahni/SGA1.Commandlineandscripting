5. RECOMMENDATIONS FOR IMPROVING STORAGE UTILIZATION
   To optimize this environment for the upcoming application environment, 
   the following storage policies are recommended:
   
   * Implement Log Rotation: Configure 'logrotate' aggressively for the application 
     logs directory to compress and purge stale records weekly, capping disk footprints.
   * Separate Storage Volumes: Mount high-write application data directories 
     (e.g., database storage or upload folders) onto isolated logical volumes (LVM). 
     This guarantees that if data overflows, core OS system services stay operational.

     to send triggers when any file system disk usage crosses 85% or Inode utilization 
     exceeds 80%.
